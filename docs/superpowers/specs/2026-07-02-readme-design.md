# 语境生词本 README 设计

- 日期：2026-07-02
- 状态：已确认，待实施

## 目标

为 GitHub 仓库提供一份中文、用户优先的根目录 `README.md`。新用户应能快速理解产品用途并完成安装；开发者应能在文档后半部分找到准确的运行、测试、构建和打包命令。

## 受众与叙事顺序

README 同时服务两类读者，但优先普通用户：

1. 普通用户：了解功能、安装扩展、授权网页、开始查词和保存。
2. 开发者：了解技术栈、本地开发、验证和打包方式。

内容先回答“它有什么用、怎么安装”，再回答“项目如何开发和验证”。

## 内容结构

1. 项目名称与一句话定位。
2. 核心功能：
   - 稳定悬停 500ms 后查词；
   - 查词成功并继续停留 2000ms 后自动保存；
   - 同一页面同一文本位置只保存一次；
   - 本地发音、生词库、每周摘要、导入导出和数据清理。
3. Chrome 与 Arc 手动安装方法。
4. 首次授权、日常使用、撤销保存和侧边栏入口。
5. 升级方法与本地数据保留注意事项。
6. 常见问题：
   - 悬停无反应；
   - 没有发音；
   - 更新后旧数据不可见；
   - Chrome 不允许直接安装 ZIP。
7. 本地开发：
   - 环境要求；
   - 安装依赖；
   - 开发、测试、类型检查、构建、E2E 和打包命令。
8. 隐私、第三方许可和当前发布状态。

## 内容约束

- 不添加尚不存在的 Chrome Web Store 链接、在线服务或正式商业承诺。
- 不虚构截图、徽章、下载量、兼容性范围或贡献者信息。
- 所有命令必须来自当前 `package.json`。
- 隐私说明与 `PRIVACY.md` 保持一致：本地优先、无广告、无分析、无遥测。
- 安装说明明确 ZIP 必须先解压，并选择包含 `manifest.json` 的目录。
- 升级说明优先覆盖原目录并重新加载，不建议删除扩展，以降低本地数据因扩展 ID 改变而不可见的风险。

## 验收标准

- 根目录存在 `README.md`。
- 普通用户无需阅读源码即可完成安装、授权和首次查词。
- 开发者可以直接复制命令完成开发、测试、构建与打包。
- README 中没有与当前代码、权限、隐私政策或产品状态矛盾的描述。
